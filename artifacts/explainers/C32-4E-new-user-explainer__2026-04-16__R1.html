ThetaFrame New-User Explainer

Five real product slides for a 60-90 second walkthrough.

Target run time: 75 seconds
Slide 1 · 12 seconds

ThetaFrame gives the week a calm operating system.

The app is organized around lanes. Each lane has one job, so users can move from orientation to action without turning the product into a noisy dashboard.

One shell Navigation, identity, and mode stay consistent across every lane.
Clear starting point New users begin with the core Daily, Weekly, and Vision surfaces.
Speaker note: Start by naming ThetaFrame as a quiet structure for deciding what matters today, this week, and over the longer arc.
Slide 2 · 18 seconds

Basic starts with Daily, Weekly, and Vision.

Basic users get the planning spine by default: today, the week, and the longer view. The AI assistant remains scoped to the lanes the user can access.

Daily Choose a realistic starting point for today.
Weekly Protect the week around theme and next steps.
Vision Keep long-range direction visible.
Speaker note: Explain that Basic is not a stripped-down admin view. It is the intended default for someone starting to use ThetaFrame.
Slide 3 · 14 seconds

Advanced lanes are assigned intentionally.

Life Ledger is an example of a Select Authorized lane. It can hold dated obligations and reminders, but it only appears when Admin assigns that access.

Not hidden-only security Route and API checks enforce the same lane permissions.
Expandable without billing logic The access framework can support future paid packaging later.
Speaker note: Keep this simple: Admin can grant advanced surfaces, and Basic still keeps the core planning loop.
Slide 4 · 16 seconds

AI creates drafts. People approve changes.

High-risk data does not silently change. AI-generated suggestions land in review surfaces first, then an allowed user approves, rejects, or applies them.

Review-first by default Drafts carry provenance and approval state before they affect user data.
Lane-scoped assistant AI operates inside the lanes the current user is allowed to use.
Speaker note: Use this slide to establish trust: the system can help populate work, but it does not bypass review for sensitive changes.
This screenshot is from production. Private card content is intentionally blurred; the visible product behavior is the review-and-apply workflow.
Slide 5 · 15 seconds

Access levels keep the app private and controlled.

The product can grow without making every feature visible to every account. Admin sees everything, Select Authorized gets assigned lanes, and Basic stays focused.

Basic User Daily, Weekly, Vision, and lane-scoped AI support.
Select Authorized Basic plus Admin-assigned modules such as Life Ledger.
Admin Everything built now, plus assignment and review tooling.
Speaker note: Close by saying the app is ready to explain to new users now, while paid packaging can be defined later on top of the same access model.
